# Snapshot Profile Save & Drive Fix — Design Spec

**Date:** 2026-03-27
**Project:** Nexus CRM (Ozsky Migration)

---

## Goal

1. Fix a misleading "folder not found" Drive error shown for new clients whose Drive folder exists but is empty.
2. Auto-save the AI-generated snapshot to the client profile whenever either snapshot button is clicked.
3. Display the saved snapshot on the Profile tab for quick consultant reference.

---

## Scope

### In scope

| Area | Change |
|------|--------|
| Drive false "not found" error | `SmartAI.jsx`: split two-branch condition into three to handle empty folder separately |
| `generateSnapshot` (.txt button, App.js) | Save snapshot text + date to `client.profile` before downloading |
| SmartAI `✨ 生成客户快照` button | Add `onSaveSnapshot` prop; auto-call after generation completes |
| Profile tab (App.js) | Display saved snapshot below 6-field grid with last-updated date |
| Data model | Add `client.profile.snapshot` (string) and `client.profile.snapshotDate` (ISO date string) |

### Out of scope

- Snapshot history / versioning
- Manual snapshot editing in the profile tab (read-only display)
- Any changes to the snapshot generation prompt or content
- Drive folder search logic (search is correct; the bug is in the status display only)
- All other tabs and pages

---

## Architecture

Two files changed:
1. **`src/SmartAI.jsx`** — Drive status fix + `onSaveSnapshot` prop
2. **`src/App.js`** — `generateSnapshot` saves to profile; `onSaveSnapshot` passed to SmartAI; profile tab renders snapshot

No new files. No API changes. No database migration (data stored in existing `client.profile` JSON blob).

---

## 1. Data Model

Two new fields stored inside `client.profile` (the existing JSON column in Supabase):

| Field | Type | Description |
|-------|------|-------------|
| `client.profile.snapshot` | string | Full snapshot text as generated by AI |
| `client.profile.snapshotDate` | string (ISO date, `YYYY-MM-DD`) | Date the snapshot was last generated |

Both are written via the existing `onSaveProfile(updatedClient)` callback, which spreads the full client object back to Supabase. No existing fields are overwritten — the save uses `{ ...client, profile: { ...p, snapshot: text, snapshotDate: date } }`.

## 2. Drive Fix (`src/SmartAI.jsx`)

**File:** `src/SmartAI.jsx`, lines ~677–771

**Problem:** The current condition `if (driveData.folderFound && driveData.processed?.length)` has only two branches. When a client folder exists but is empty (no processable files), `folderFound` is `true` but `processed.length` is `0`, so it falls into the `else` branch, which calls `setDriveStatus({ found: false, message: driveData.message })`. Since the API only sets `message` when `folderFound` is `false`, `driveData.message` is `undefined`, and SmartAI renders the misleading fallback `'ozsky-clients 文件夹中未找到该客户文件夹'`.

**Fix:** Replace the two-branch condition with three explicit cases:

```js
if (!driveData.folderFound) {
  // Case 1: folder genuinely not found in Drive
  setDriveStatus({ found: false, message: driveData.message });
} else if (!driveData.processed?.length) {
  // Case 2: folder found but empty or all files skipped
  setDriveStatus({ found: true, folderName: driveData.folderName, fileCount: driveData.totalFiles || 0, readCount: 0, fileDebug: [] });
} else {
  // Case 3: folder found with readable files — existing processing path
  // ... (unchanged file-processing loop)
  setDriveStatus({ found: true, folderName: driveData.folderName, fileCount: driveData.totalFiles, readCount: textParts.length + pdfBlocks.length, fileDebug });
}
```

Case 2 sets `found: true` so the status bar renders the green "✅ folder found" variant with `readCount: 0`.

## 3. Snapshot Auto-Save

### 3a. `generateSnapshot` (App.js)

**File:** `src/App.js`, `generateSnapshot` function (~line 1186)

**Current behaviour:** Generates snapshot text → downloads `.txt` file.

**Change:** After `snapshotText` is confirmed non-empty, save to profile before downloading:

```js
const today = new Date().toISOString().slice(0, 10);
await onSaveProfile({
  ...client,
  profile: { ...(client.profile || {}), snapshot: snapshotText, snapshotDate: today },
});
// existing download code follows unchanged
```

The `today` variable replaces the existing `toLocaleDateString` string for the data field (the `.txt` download still uses the human-readable date for the document content). The save is `await`-ed before the download so any save error surfaces cleanly.

### 3b. SmartAI `✨ 生成客户快照` (SmartAI.jsx + App.js)

**New prop on `SnapshotSection` component (SmartAI.jsx):**

```js
function SnapshotSection({ ..., onSaveSnapshot }) {
  ...
  const generate = async () => {
    ...
    setSnapshot(snapshotText);
    onSaveSnapshot?.(snapshotText);  // ← add this line after setSnapshot
  };
```

The prop is optional (`?.`) so existing usages without it remain safe.

**Pass the prop from App.js** (in the `{tab === 'ai' && ...}` block where `<SmartAI>` is rendered, and `SmartAI` passes it down to `SnapshotSection`):

In App.js, add `onSaveSnapshot` to the `<SmartAI>` element:
```jsx
<SmartAI
  selectedClient={client}
  selectedCase={clientJobs[0] || null}
  onImportClient={...}
  onImportCase={...}
  onAddNote={...}
  onSaveSnapshot={(text) => {
    const date = new Date().toISOString().slice(0, 10);
    onSaveProfile({ ...client, profile: { ...(client.profile || {}), snapshot: text, snapshotDate: date } });
  }}
/>
```

In SmartAI.jsx, thread `onSaveSnapshot` from the exported `SmartAI` component down to `SnapshotSection`:
- `SmartAI` receives `onSaveSnapshot` as a prop
- Passes it to `<SnapshotSection onSaveSnapshot={onSaveSnapshot} ...>`
- `SnapshotSection` calls `onSaveSnapshot?.(snapshotText)` after `setSnapshot`

## 4. Profile Tab Snapshot Display (App.js)

**File:** `src/App.js`, profile tab block, after the 6-field grid closing `</div>` (~line 1385)

Add a snapshot section that renders only when `p.snapshot` exists:

```jsx
{p.snapshot && (
  <div style={{ marginTop:20, borderTop:'1.5px solid #e2e8f0', paddingTop:16 }}>
    <div style={{ display:'flex', justifyContent:'space-between', alignItems:'center', marginBottom:8 }}>
      <span style={{ fontSize:11, fontWeight:700, color:'#6366f1', textTransform:'uppercase', letterSpacing:'0.07em' }}>
        客户快照
      </span>
      {p.snapshotDate && (
        <span style={{ fontSize:11, color:'#9ca3af' }}>更新于 {p.snapshotDate}</span>
      )}
    </div>
    <div style={{ background:'#f8fafc', border:'1px solid #e2e8f0', borderRadius:8, padding:12,
      fontSize:12, color:'#374151', lineHeight:1.75, whiteSpace:'pre-wrap',
      maxHeight:400, overflowY:'auto',
      fontFamily:"'JetBrains Mono','Courier New',monospace" }}>
      {p.snapshot}
    </div>
  </div>
)}
```

Where `p = client.profile || {}` (already defined in `ClientDetailModal`).

## Edge Cases

| Case | Behaviour |
|------|-----------|
| Client has no snapshot yet | Snapshot section hidden entirely from profile tab |
| `onSaveProfile` fails | Error dispatched via existing `ozsky-db-error` event (or `window.alert` in `generateSnapshot`) |
| SmartAI used without `onSaveSnapshot` prop | `onSaveSnapshot?.()` is a no-op — no crash |
| Snapshot generated twice | Second save overwrites `snapshot` and `snapshotDate` — always shows latest |
